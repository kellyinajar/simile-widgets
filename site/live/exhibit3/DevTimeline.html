Development Schedule for Exhibit 3.0 Project

August

  • Provide a server-side option for publishing large collections
  • Support links to particular Exhibit states
  • Initial version of method to move from small to large collections stored on the server
  • API for result set pagination and/or summarization (scripted)

September

  • Improved embed-ability of Exhibits into external applicatins and Web pages

October

  • Scalabe to 1 million triples and 20,000 facet values
  • Data export features enabling large-scale data editing in external applicatons (i.e. Google Refine)

November

  • API for result set pagination and/or summarization (staged)

December

  • Incremental updates of UI views supported
  • Easier way to author new types of views, widgets, and facets
